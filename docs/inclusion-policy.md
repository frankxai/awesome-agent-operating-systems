# Inclusion Policy

This list is curated, not exhaustive.

## Strong Fit

- Agent runtimes with tools, memory, execution, or orchestration.
- Coding agents that can inspect and change repos.
- MCP servers and tool protocols.
- Agent memory, provenance, eval, safety, or observability systems.
- Deployment platforms commonly used for agent surfaces.
- Managed products that clearly expose agentic development or operation workflows.

## Weak Fit

- Model-only pages.
- Prompt collections without operational behavior.
- Closed products with unclear capabilities.
- Hype pages that cannot be verified through docs, product pages, or repositories.

## Entry Format

Each entry should answer:

- What is it?
- What job does it help with?
- Is it local, cloud, managed, or hybrid?
- What is the primary source link?

## Removal

Entries may be removed if the project is abandoned, misleading, hostile to users, or no longer relevant to agent operations.
